# WP Multi AI Writer

**🤖WP AI 助手** 是一个为 WordPress 经典编辑器（Classic Editor）设计的 AI 内容生成插件。它允许您通过接入 DeepSeek、通义千问（Qwen）、豆包（Doubao）等主流 AI 大模型，一键生成 SEO 友好的高质量文章。

## 功能特点

- **多模型支持**：集成 DeepSeek、通义千问、豆包 API，可在后台自由切换。
- **经典编辑器集成**：在文章编辑页添加“AI 生成文章”按钮，弹出模态框进行生成。
- **文章长度控制**：支持短文（100字）、中长（200字）、长文（500字）、超长（1000字）。
- **写作风格选择**：专业正式、轻松口语、热情感染。
- **API Key 测试**：在设置页面可一键测试 API Key 是否有效。
- **同步生成**：提交后等待 AI 返回完整内容，自动插入到编辑器光标位置（支持文本模式和可视化模式）。
- **标题自动填充**：若文章标题为空且 AI 返回了标题，会自动填入标题栏。
- **安全高效**：所有请求均通过 WordPress REST API 进行，支持 Nonce 验证和权限控制。

## 系统要求

- WordPress 5.0 或更高版本
- PHP 7.2 或更高版本
- 经典编辑器插件（可选，若不使用古腾堡，推荐安装 Classic Editor）
- 有效的 AI 模型 API Key（DeepSeek / 通义千问 / 豆包）

## 安装步骤

### 手动安装

1. 下载插件压缩包 `wp-multi-ai-writer.zip`。
2. 登录 WordPress 后台，进入 **插件 → 安装插件 → 上传插件**。
3. 选择压缩包并点击“立即安装”。
4. 安装完成后，点击“启用插件”。

### 通过 FTP 安装

1. 解压插件包，将 `wp-multi-ai-writer` 文件夹上传到 `/wp-content/plugins/` 目录。
2. 登录 WordPress 后台，进入 **插件** 页面，找到 **WP Multi AI Writer** 并点击“启用”。

## 配置说明

### 1. 进入设置页面

启用插件后，左侧菜单会出现 \*_🤖 AI 助手设置_ 图标，点击进入设置页面。

### 2. 通用设置

| 选项           | 说明                                          | 推荐值   |
| -------------- | --------------------------------------------- | -------- |
| 默认模型       | 在生成文章时默认选择的 AI 模型                | DeepSeek |
| 请求超时（秒） | API 请求的最大等待时间                        | 60       |
| 最大 Token 数  | 控制生成内容的最大长度（1 token ≈ 0.75 汉字） | 2000     |
| 创意程度       | 数值越高，生成内容越有创意（0~1）             | 0.7      |

### 3. 模型 API 配置

为每个需要使用的模型填写对应的 API Key，并点击“测试”按钮验证有效性。

- **DeepSeek**：[获取 API Key](https://platform.deepseek.com/)
- **通义千问**：[阿里云百炼平台](https://bailian.console.aliyun.com/)
- **豆包**：[火山引擎方舟平台](https://console.volcengine.com/ark/)

> 至少配置一个有效的 API Key 才能使用生成功能。

### 4. 保存设置

填写完成后，点击 **保存所有设置**。

## 使用方法

### 生成文章

1. 新建或编辑一篇**经典编辑器**文章（若使用古腾堡编辑器，请切换到经典编辑器模式）。
2. 在文章标题下方，点击 **🤖 AI 生成文章** 按钮。
3. 在弹出的模态框中：
   - 输入文章主题（例如：“WordPress 性能优化技巧”）
   - 选择 AI 模型
   - 选择写作风格
   - 选择目标长度
4. 点击 **开始生成**。
5. 等待 10~60 秒（取决于模型和网络），生成的内容将自动插入到编辑器当前光标位置。
6. 如果标题栏为空，AI 返回的标题会自动填入。

### 手动调整

生成后，您可以对文章进行人工修改、添加图片或进一步 SEO 优化。

## 常见问题

### 1. 点击“开始生成”后没有反应或提示“提交失败”

- 确保已正确配置 API Key 并测试通过。
- 检查浏览器的控制台（F12 → Console）是否有红色错误。
- 确认当前使用的是**经典编辑器**（非古腾堡）。
- 尝试刷新页面后重试。

### 2. 生成的内容没有插入到编辑器

- 如果您使用的是可视化模式（TinyMCE），插件会自动插入。如果失败，请切换到文本模式再试。
- 检查浏览器控制台是否有 “未找到编辑器元素” 的提示。

### 3. 生成的内容不符合主题或质量差

- 尝试调整“创意程度”参数。
- 更换其他 AI 模型测试。
- 优化主题描述，使其更具体（例如：“为新手写一篇关于 WordPress 安全插件的推荐文章”）。

### 4. API 测试时提示“网络错误”

- 检查服务器是否能正常访问外网（可联系主机商）。
- 临时关闭防火墙或安全组。
- 增加“请求超时”值。

### 5. 菜单“AI Writer”不显示

- 停用并重新激活插件。
- 确认当前用户是管理员。
- 检查是否有其他插件冲突。

## 高级自定义

### 修改文章长度选项

编辑 `/includes/class-editor-integration.php`，找到长度下拉框的 HTML，修改 `value` 和显示文本即可。

### 添加其他 AI 模型

1. 在 `/api-providers/` 目录下创建新的 Provider 类，实现 `WMAIW_AI_Provider` 接口。
2. 在 `/includes/class-settings.php` 中的 `wmaiw_models` 部分添加新模型的 API Key 字段。
3. 在 `/includes/class-api-manager.php` 中加载新类。
